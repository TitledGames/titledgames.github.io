# Documentation Index

Welcome to the Titled Games website documentation! This folder contains all the guides and resources you need to contribute to our project.

## Available Guides

### For Complete Beginners

**Start here if you're new to web development or contributing to open source:**

1. **[Setup Guide](SETUP.md)** - First-time setup instructions
   - Install required tools (Git, text editor)
   - Clone the repository
   - Get your development environment ready
   - ~20 minutes to complete

2. **[Beginner's Guide](BEGINNER_GUIDE.md)** - Learn the basics
   - HTML fundamentals
   - CSS styling
   - JavaScript interactivity
   - How our website is structured
   - Making your first changes
   - ~1-2 hours to read through

3. **[Quick Reference](QUICK_REFERENCE.md)** - Common tasks and code snippets
   - CSS color changes
   - HTML structure additions
   - JavaScript event handlers
   - Debugging tips
   - Keep this open while coding!

### For All Contributors

**Everyone should read these:**

4. **[Contributing Guide](../CONTRIBUTING.md)** - How to contribute
   - Development workflow
   - Git basics
   - Code style guidelines
   - Testing checklist
   - Submitting pull requests

5. **[Main README](../README.md)** - Project overview
   - What is Titled Games
   - Features and technology

## Learning Path

### Path 1: Complete Beginner
**"I've never done web development before"**

1. Read [SETUP.md](SETUP.md) → Set up your environment
2. Read [BEGINNER_GUIDE.md](BEGINNER_GUIDE.md) → Learn the basics
3. Read [CONTRIBUTING.md](../CONTRIBUTING.md) → Understand workflow
4. Open `index.html` and follow along with examples
5. Make a small test change
6. Pick a `good first issue` on GitHub
7. Use [QUICK_REFERENCE.md](QUICK_REFERENCE.md) as needed

**Time**: 3-5 hours to get comfortable

### Path 2: Some Experience
**"I know a bit of HTML/CSS but never used Git"**

1. Skim [BEGINNER_GUIDE.md](BEGINNER_GUIDE.md) → Review basics
2. Read [SETUP.md](SETUP.md) → Focus on Git setup
3. Read [CONTRIBUTING.md](../CONTRIBUTING.md) → Learn our workflow
4. Clone the repository and run it locally
5. Find an issue to work on
6. Reference [QUICK_REFERENCE.md](QUICK_REFERENCE.md) as needed

**Time**: 1-2 hours to get started

### Path 3: Experienced Developer
**"I'm comfortable with web dev and Git"**

1. Read [README.md](../README.md) → Understand the project
2. Skim [CONTRIBUTING.md](../CONTRIBUTING.md) → Check our conventions
3. Clone and run locally
4. Browse the code
5. Pick any issue or suggest improvements
6. Use [QUICK_REFERENCE.md](QUICK_REFERENCE.md) for project-specific patterns

**Time**: 15-30 minutes to get oriented

## Tips for Using This Documentation

1. **Start with the basics**: Don't skip setup steps
2. **Keep Quick Reference handy**: Bookmark it for easy access
3. **Don't read everything at once**: Come back as needed
4. **Try examples**: The best way to learn is by doing
5. **Ask questions**: Use Discord or GitHub if confused
6. **Contribute to docs**: Found something unclear? Submit improvements!

## Still Need Help?

If you can't find what you're looking for:

1. **Search the docs**: Use Ctrl+F (Cmd+F on Mac) to search
2. **Check existing issues**: Someone may have asked already
3. **Ask in Discord**: [dsc.gg/titledgames](https://dsc.gg/titledgames)
4. **Open an issue**: [GitHub Issues](https://github.com/TitledGames/website/issues)
5. **Start a discussion**: [GitHub Discussions](https://github.com/TitledGames/website/discussions)

## Keeping Updated

Documentation is regularly updated. To stay current:

- **Watch the repository** on GitHub for notifications
- **Check commit history** to see recent changes
- **Pull latest changes** regularly: `git pull upstream main`
- **Join Discord** for announcements

## Contributing to Documentation
