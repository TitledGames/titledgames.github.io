# Titled Games Website

Welcome to the official repository for the Titled Games website! This is where we build and maintain our web presence, showcasing our indie game studio and the games we create.

## Quick Start

### For Complete Beginners

**New to web development?** Start here:
1. Read our [Beginner's Guide](docs/BEGINNER_GUIDE.md) to learn HTML, CSS, and JavaScript basics
2. Follow the [Contributing Guide](CONTRIBUTING.md) to make your first contribution
3. Open `index.html` in any web browser to see the website locally

### For Developers

1. **Clone the repository**
   ```bash
   git clone https://github.com/TitledGames/website.git
   cd website
   ```

2. **Open in your browser**
   - Simply open `index.html` in your web browser
   - No build process or dependencies required!

3. **Make changes**
   - Edit `index.html` for content
   - Edit `styles.css` for styling
   - Edit `script.js` for functionality
   - Refresh your browser to see changes

4. **Submit your contribution**
   - Create a branch
   - Make your changes
   - Submit a pull request

See [CONTRIBUTING.md](CONTRIBUTING.md) for detailed instructions.

## Contributing

We welcome contributions from everyone, whether you're fixing a typo or adding a major feature!

### Ways to Contribute

1. **Report Bugs**: Found something broken? [Open an issue](https://github.com/TitledGames/website/issues)
2. **Suggest Features**: Have an idea? [Start a discussion](https://github.com/TitledGames/website/discussions)
3. **Improve Documentation**: Help make our guides even better
4. **Fix Issues**: Check out [open issues](https://github.com/TitledGames/website/issues) and submit a PR
5. **Add Games**: Showcase new games on the website

### Getting Started

1. **Read the guides**
   - [Beginner's Guide](docs/BEGINNER_GUIDE.md) - Learn web development basics
   - [Contributing Guide](CONTRIBUTING.md) - Detailed contribution instructions

2. **Pick an issue**
   - Look for issues labeled `good first issue` or `help wanted`
   - Or create your own feature/fix

3. **Make your changes**
   - Fork the repository
   - Create a branch
   - Make your changes
   - Test thoroughly

4. **Submit a Pull Request**
   - Push to your fork
   - Open a PR with a clear description
   - Wait for review
